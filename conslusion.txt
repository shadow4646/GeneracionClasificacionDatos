1. LO APRENDIDO DURANTE EL DESARROLLO DEL PROYECTO
========================================================================

Durante el desarrollo de este proyecto se adquirieron y reforzaron varios
conocimientos fundamentales de programacion en Java:

- Manejo de archivos de texto plano en Java mediante las clases BufferedReader,
  BufferedWriter, FileReader y FileWriter, comprendiendo la importancia de
  cerrar correctamente los recursos con el bloque try-with-resources.

- Uso de la clase Random para la generacion de datos pseudoaleatorios coherentes,
  garantizando que los valores generados cumplan restricciones logicas como
  precios positivos e IDs validos.

- Implementacion de estructuras de datos como HashMap para almacenar y relacionar
  informacion de vendedores y productos de manera eficiente.

- Ordenamiento de colecciones con Collections.sort y Comparator, aplicado al
  ranking de vendedores por ventas y productos por cantidad vendida.

- Documentacion del codigo siguiendo el estandar JavaDoc, incluyendo descripcion
  de clases, parametros, retornos y excepciones.

- Organizacion de un proyecto en Eclipse para Java SE 8 y gestion del codigo
  fuente mediante control de versiones con Git y GitHub.

- Validacion de datos de entrada para detectar formatos incorrectos e informacion
  incoherente, como IDs de productos inexistentes o cantidades negativas.

- Procesamiento de multiples archivos por vendedor: la clase main acumula los
  totales de ventas usando el numero de documento como clave, por lo que varios
  archivos que pertenecen al mismo vendedor se agregan correctamente sin duplicar
  registros en el reporte final.

========================================================================
2. POSIBLES APLICACIONES EN LA VIDA PROFESIONAL
========================================================================

Las destrezas adquiridas en este proyecto tienen aplicacion directa en
multiples contextos del entorno profesional:

- Procesamiento de archivos planos y CSV: muchas empresas intercambian datos
  entre sistemas mediante archivos de texto. Saber leerlos, validarlos y
  transformarlos es una habilidad de uso cotidiano en roles de desarrollo
  backend, analisis de datos e integracion de sistemas.

- Generacion de reportes automatizados: la capacidad de tomar datos en bruto
  y producir informes ordenados y clasificados es fundamental en sistemas de
  gestion empresarial, plataformas de comercio electronico y herramientas
  de business intelligence.

- Validacion y calidad de datos: la deteccion de datos incoherentes o mal
  formateados es critica en cualquier sistema que procese informacion de
  terceros, como integraciones con proveedores, importaciones masivas o
  migraciones de bases de datos.

- Control de versiones con Git: el uso de repositorios para gestionar el
  codigo fuente es una practica estandar en la industria del software, y
  la experiencia adquirida facilita el trabajo en equipo en proyectos reales.

========================================================================
3. DIFICULTADES PRESENTADAS DURANTE EL DESARROLLO
========================================================================

A lo largo del proyecto se presentaron las siguientes dificultades:

- Garantizar coherencia entre archivos: asegurar que los IDs y nombres de
  los vendedores fueran identicos en el archivo de informacion y en cada
  archivo individual de ventas requirio pre-generar los datos en memoria
  antes de escribirlos, en lugar de generarlos de forma independiente.

- Manejo del formato de los archivos: las lineas de ventas incluyen un punto
  y coma al final (IDProducto;Cantidad;), lo que requirio un tratamiento
  especial al momento de parsear para evitar errores al dividir los campos.

- Identificacion de archivos de ventas: distinguir automaticamente los archivos
  de ventas de los demas archivos .txt presentes en la carpeta (productos,
  vendedores, conclusiones) sin solicitar informacion al usuario implico definir
  una lista de exclusiones y confiar en las convenciones de nombre de archivo.

- Soporte de multiples archivos por vendedor (extra a): garantizar que al generar
  un segundo archivo para el mismo vendedor, el encabezado llevara el mismo
  TipoDocumento y NumeroDocumento para que la clase main pudiera reconocerlo y
  acumular las ventas correctamente.

- Compatibilidad con Java 8: algunas construcciones modernas de Java (como
  lambdas y streams) no podian usarse de la misma forma que en versiones mas
  recientes, por lo que fue necesario utilizar clases anonimas para los
  Comparator y tipos genericos explicitos en las declaraciones de colecciones.
